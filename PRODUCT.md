# Product

## Register

product

## Users

Solo developers building and iterating on AI agent workflows in focused, self-directed sessions. They work alone, often late or in deep focus, and need a tool that stays out of the way while giving them full control over multi-agent orchestration. They switch between configuring providers, building flows, and monitoring executions — often within the same session.

## Product Purpose

ForkFlow is an open-source, self-hosted AI agent orchestration platform with a visual node-based flow editor. It lets developers design, run, and debug multi-agent workflows through a drag-and-drop interface — connecting conversation nodes, processor nodes, and conditional routes into executable pipelines. Success means a developer can go from zero to a working multi-agent flow in minutes, with full visibility into each step.

## Brand Personality

Clever, fast, approachable. The interface feels smart without being clever-for-its-own-sake — every interaction is purposeful. Speed is tangible: responsive controls, instant feedback, fluid canvas manipulation. Approachable means no intimidating complexity on the surface; power reveals itself as you dig in, not as a wall upfront.

Visual tone: inspired by Anthropic's website — refined dark theme, generous whitespace, confident typography, smooth micro-interactions. Easy on the eyes, satisfying to use, never cluttered. Monochromatic palette: no accent color splashes, hierarchy through luminance alone. Sharp corners only (0 radius) — no rounded edges, no soft shapes. The aesthetic is architectural, not decorative.

## Anti-references

- SaaS template clichés: gradient heroes, glassmorphism cards, cream/sand backgrounds, metric dashboards with big numbers
- Langflow/n8n visual clutter: dense toolbars, cramped panels, 12-color node palettes
- Overly technical terminal aesthetics: green-on-black, monospace-only, no visual hierarchy
- Playful/casual UI that undermines the sense of a serious tool

## Design Principles

1. **Fluid over fragmented** — Transitions between views and states should feel seamless. No jarring reloads, no disconnected panels. The canvas, properties, and logs form one continuous experience.
2. **Progressive disclosure** — Show what's needed now, reveal depth on demand. Configuration power doesn't require upfront complexity. Empty states teach; advanced options fold away.
3. **Quiet confidence** — The interface doesn't announce itself. No decorative elements, no visual noise. Every pixel earns its place. Refined restraint, not minimal-for-minimal's-sake.
4. **Responsive craft** — Every interaction gives tactile feedback: button presses, drag snaps, panel slides, status transitions. The tool feels alive and precise.
5. **Dark-first readability** — The dark theme is the primary experience. Text contrast is non-negotiable. Hierarchy comes from luminance, not color saturation.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum; aim for AAA on body text contrast
- `prefers-reduced-motion` respected for all animations
- Keyboard navigation for flow editor and forms
- Color is never the sole indicator of state
